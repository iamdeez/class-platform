| 환경 | 목적 | 비고 |
|---|---|---|
| dev (로컬) | 개발·학습 목적, Docker Compose로 로컬 MySQL 기동 예정 (001 spec T002) | 아직 파일 미생성 |
| staging / prod | 없음 | 005 spec(CI/CD·배포)에서 계획 예정. 연습 프로젝트 특성상 실제 운영 환경은 없을 수 있음 |

## 2. 인프라 토폴로지

**현재 실제 상태**: 인프라 구성 파일(`docker-compose.yml` 등)이 아직 없다.

**계획** (`docs/specs/v0.1.0/001-class-enrollment-core/plan.md` 기준, 미구현):

```
[Spring Boot App] ──→ [MySQL 컨테이너]
```

002/003 spec에서 MongoDB(커뮤니티), Redis(캐싱)가 추가될 예정이다.

## 3. 배포 방식

[미파악] — 005 spec(테스트 강화·CI/CD·배포 자동화)에서 GitHub Actions 기반 CI/CD, AWS 배포를 계획 중이나 아직 미착수.

## 4. 모니터링·로깅

[미파악] — 채용공고에서 Pinpoint/Sentry가 언급되나, 이 연습 프로젝트에 도입할지는 미결정. 도입 시 별도 spec으로 설계한다.

## 5. 연결 실패 재시도 동작

[미파악] — 외부 시스템 연동이 아직 없음 (001 spec은 MySQL 단일 의존).
